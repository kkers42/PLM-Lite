# PLM Lite v1.1.0 — Release Notes

## Overview
v1.1.0 adds NX workstation agent integration — a lightweight bridge between NX 12 and the PLM server that enables automatic session tracking, in-place file opens, and NX-triggered checkout/checkin.

---

## New Features

### Workstation Agent (`workstation_agent/`)
A small Flask app that runs on `localhost:27370` on each NX workstation.
- Registers the workstation as an active PLM session on startup
- Handles `plmopen://` URI scheme — opens the correct CAD file in NX directly from the PLM web UI
- Forwards checkout and checkin actions from NX hooks to the PLM server
- Installer (`install.ps1`) sets up the agent, registers the URI scheme, and creates a Start Menu shortcut

### NX Open Hook (`nx_hook/`)
A Python journal placed in the NX 12 startup folder.
- Pings PLM on NX startup and file open to maintain an active session
- Calls checkin on file save if the part is currently checked out by this user
- Completely silent on error — never blocks NX

### Cache Sessions (`cache_sessions` table)
Tracks which workstations have an active NX session connected to PLM.
- Admin UI endpoint: `GET /api/cache/sessions`
- Close a stale session: `DELETE /api/cache/sessions/{id}`
- Agent keep-alive: `POST /api/cache/ping`

### Manifest Sidecars (`manifest_sidecars` table)
Stores cached assembly manifest metadata alongside CAD documents.
- `GET /api/parts/{id}/manifest` — retrieve manifest for a part
- `POST /api/parts/{id}/manifest` — create or update manifest

## Upgrade Notes
- Run `database.initialize()` on startup applies the new tables automatically (uses `CREATE TABLE IF NOT EXISTS`)
- No data migration required — existing parts, documents, and users are unaffected
- Workstation agent is optional — PLM continues to work normally without it
